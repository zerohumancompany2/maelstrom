## Test Descriptions

### Test 1: TestPrepareContextForBoundary_StripsForbidden

**Given**: A runtime with ContextMap containing blocks with taints `["INNER_ONLY", "SECRET", "TOOL_OUTPUT"]` and target boundary `dmz`  
**When**: PrepareContextForBoundary is called with boundary `dmz`  
**Then**: Blocks with `INNER_ONLY` and `SECRET` taints are stripped, `TOOL_OUTPUT` blocks preserved  
**Expected Result**: Forbidden taints removed for DMZ boundary, allowed taints retained  
**Spec Reference**: arch-v1.md L292-296 (ContextMap Assembly Enforcement), L284 (taint stripping)

### Test 2: TestPrepareContextForBoundary_EnforcesPolicy

**Given**: A runtime with ContextMap containing blocks with `taintPolicy: {enforcement: "strict", redactRules: [{taint: "PII", replacement: "[REDACTED]"}]}` and data with taints `["PII"]`  
**When**: PrepareContextForBoundary is called with boundary `outer`  
**Then**: PII-tainted values are redacted according to redactRules, blocks remain in context  
**Expected Result**: TaintPolicy enforcement applied, redaction rules executed  
**Spec Reference**: arch-v1.md L292-296 (ContextMap Assembly Enforcement), L218-221 (redactRules)

### Test 3: TestPrepareContextForBoundary_ReturnsEmpty

**Given**: A runtime with ContextMap containing only blocks with taints `["INNER_ONLY", "SECRET"]` and `allowedForBoundary: ["inner"]`  
**When**: PrepareContextForBoundary is called with boundary `outer`  
**Then**: All blocks are filtered out, empty context returned  
**Expected Result**: No blocks visible to outer boundary when all marked inner-only  
**Spec Reference**: arch-v1.md L292-296 (ContextMap Assembly Enforcement), L223 (allowedForBoundary)
