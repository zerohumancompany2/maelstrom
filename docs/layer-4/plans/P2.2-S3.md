## Test Descriptions

### Test 1: TestS3DataSource_TagOnWrite

**Given**: An S3 DataSource configured with bucket "my-bucket" and region "us-east-1"  
**When**: TagOnWrite is called with key "documents/secret.txt" and taints ["SECRET", "INNER_ONLY"]  
**Then**: S3 PutObjectTagging is called with correct tag set  
**Expected Result**: Object tags set on S3 object, taints stored as tag values with key "maelstrom-taints"  
**Spec Reference**: arch-v1.md L758 (object storage tags), L771 (TagOnWrite workflow)

### Test 2: TestS3DataSource_GetTaints

**Given**: An S3 object with existing tags including "maelstrom-taints=PII,TOOL_OUTPUT"  
**When**: GetTaints is called with the object key  
**Then**: Returns ["PII", "TOOL_OUTPUT"] as decoded taint list  
**Expected Result**: Object tags read and decoded, taint list returned from tags  
**Spec Reference**: arch-v1.md L777-778 (getTaints call), L758 (object metadata)

### Test 3: TestS3DataSource_ValidateAccess

**Given**: An S3 DataSource with allowedForBoundary=["inner", "dmz"]  
**When**: ValidateAccess is called with boundary "outer"  
**Then**: Error returned indicating unauthorized boundary access  
**Expected Result**: Boundary validation works, error on unauthorized boundary  
**Spec Reference**: arch-v1.md L789 (ValidateAccess interface), L760 (allowedForBoundary)

### Test 4: TestS3DataSource_Config

**Given**: A DataSource config with bucket, region, and endpoint fields  
**When**: NewS3DataSource is called with the config  
**Then**: S3 client initialized with correct bucket name and region  
**Expected Result**: Bucket name from config, region from config, endpoint optional  
**Spec Reference**: arch-v1.md L762-765 (DataSource YAML config), implementation-assessment.md L122
