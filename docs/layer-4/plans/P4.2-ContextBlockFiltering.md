## Acceptance Criteria

1. `prepareContextForBoundary` filters blocks with forbidden taints for target boundary
2. Blocks with `redactMode: redact` have forbidden values replaced
3. Blocks with `redactMode: dropBlock` are removed from context entirely
4. Per-block `taintPolicy` overrides global policy settings
5. `allowedForBoundary` restricts block visibility to specified boundaries
6. `redactMode: audit` allows blocks through with violation logging

## Test Descriptions

### Test 1: TestPrepareContextForBoundary_Filter

**Given**: A runtime with ContextMap containing blocks with taints `["INNER_ONLY", "TOOL_OUTPUT"]` and target boundary `dmz`  
**When**: PrepareContextForBoundary is called with boundary `dmz`  
**Then**: Blocks with `INNER_ONLY` taint are filtered out, `TOOL_OUTPUT` blocks preserved  
**Expected Result**: Forbidden blocks removed, allowed blocks retained in context  
**Spec Reference**: arch-v1.md L293 (prepareContextForBoundary), L294 (forbidden taint filtering)

### Test 2: TestFilterContextBlock_Redact

**Given**: A ContextBlock with `taintPolicy.redactMode: redact` and data containing taints `["SECRET"]` with `redactRules: [{taint: "SECRET", replacement: "[REDACTED]"}]`  
**When**: FilterContextBlock is called with boundary `outer`  
**Then**: SECRET-tainted values replaced with `[REDACTED]`, block structure preserved  
**Expected Result**: Forbidden taint values redacted, block remains in context  
**Spec Reference**: arch-v1.md L295 (redact before LLM call), L218-221 (redactRules)

### Test 3: TestFilterContextBlock_DropBlock

**Given**: A ContextBlock with `taintPolicy.redactMode: dropBlock` and data containing taints `["PII"]`  
**When**: FilterContextBlock is called with boundary `outer`  
**Then**: Entire block removed from context, not present in output  
**Expected Result**: Block completely dropped when redactMode is dropBlock  
**Spec Reference**: arch-v1.md L295 (drop before LLM call), L218 (redactMode: dropBlock)

### Test 4: TestFilterContextBlock_PerBlockOverride

**Given**: A ContextBlock with per-block `taintPolicy: {redactMode: "audit"}` and global policy `enforcement: strict`  
**When**: FilterContextBlock is called with boundary `dmz`  
**Then**: Per-block audit mode applies, allowing block through with logging (overrides global strict)  
**Expected Result**: Per-block policy takes precedence over global settings  
**Spec Reference**: arch-v1.md L296 (per-block overrides global), spec-extraction.md L121

### Test 5: TestFilterContextBlock_AllowedForBoundary

**Given**: A ContextBlock with `taintPolicy.allowedForBoundary: ["inner"]` and data with taints `["SECRET"]`  
**When**: FilterContextBlock is called with boundary `dmz`  
**Then**: Block filtered out because dmz not in allowedForBoundary list  
**Expected Result**: Block only visible to boundaries in allowedForBoundary  
**Spec Reference**: arch-v1.md L223 (allowedForBoundary), spec-extraction.md L173-176

### Test 6: TestFilterContextBlock_Audit

**Given**: A ContextBlock with `taintPolicy.redactMode: audit` and data containing taints `["PII"]`  
**When**: FilterContextBlock is called with boundary `outer`  
**Then**: Block passes through unchanged, violation logged to audit trail  
**Expected Result**: No filtering, violation recorded for observability  
**Spec Reference**: arch-v1.md L218 (redactMode: audit), spec-extraction.md L286 (violation emission)
