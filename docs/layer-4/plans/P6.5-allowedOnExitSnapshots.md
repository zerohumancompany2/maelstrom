## Test Descriptions

### Test 1: TestSnapshot_AllowedOnExit_EnforcesPolicy

**Given**: A runtime with state containing taints `["TOOL_OUTPUT", "USER_SUPPLIED", "PII"]` and ChartDefinition with `allowedOnExit: ["TOOL_OUTPUT", "USER_SUPPLIED"]`  
**When**: PersistenceService.Snapshot is called for the runtime  
**Then**: Snapshot succeeds and stores taints with the snapshot record  
**Expected Result**: SnapshotRecord created with State and Taints fields populated; no error returned  
**Spec Reference**: arch-v1.md L129 (allowedOnExit definition), L280 (snapshot uses allowedOnExit), L1317 (SHALL enforce on snapshots)

### Test 2: TestSnapshot_AllowedOnExit_BlocksForbiddenTaints

**Given**: A runtime with state containing taints `["TOOL_OUTPUT", "INNER_ONLY"]` and ChartDefinition with `allowedOnExit: ["TOOL_OUTPUT", "USER_SUPPLIED"]` (INNER_ONLY is forbidden)  
**When**: PersistenceService.Snapshot is called for the runtime  
**Then**: Snapshot operation returns error indicating forbidden taint "INNER_ONLY" is not allowed  
**Expected Result**: Error returned with message indicating forbidden taint; no SnapshotRecord created  
**Spec Reference**: arch-v1.md L1363 (PII/SECRET/INNER_ONLY blocked example), L1317 (SHALL enforce on snapshots), L1518 (enforce on all snapshots)
