## Test Descriptions

### Test 1: TestSanitizeStreamChunk_PII

**Given**: A StreamChunk with content containing PII taint `["PII", "USER_SUPPLIED"]`  
**When**: SanitizeStreamChunk is called with policy that forbids PII  
**Then**: PII taint is stripped from chunk.Taints and content is redacted  
**Expected Result**: Chunk.Taints contains only `["USER_SUPPLIED"]`, PII content replaced with `[REDACTED]`  
**Spec Reference**: arch-v1.md L298 (PII redaction in DMZ sanitizers)

### Test 2: TestSanitizeStreamChunk_LengthCap

**Given**: A StreamChunk with content exceeding max length (e.g., 10000 bytes when cap is 4096)  
**When**: SanitizeStreamChunk is called with length cap policy  
**Then**: Chunk content is truncated to max length and isFinal is set to true  
**Expected Result**: Content length ≤ max length, isFinal=true to indicate truncation  
**Spec Reference**: arch-v1.md L298 (length caps in DMZ sanitizers)

### Test 3: TestSanitizeStreamChunk_Schema

**Given**: A StreamChunk with invalid schema (missing required fields or wrong types)  
**When**: SanitizeStreamChunk is called with schema validation enabled  
**Then**: Error is returned and chunk is rejected  
**Expected Result**: Error message indicates schema validation failure, chunk not emitted  
**Spec Reference**: arch-v1.md L298 (schema validation in DMZ sanitizers)

### Test 4: TestSanitizeStreamChunk_InnerData

**Given**: A StreamChunk with INNER_ONLY taint attempting to cross to DMZ boundary  
**When**: SanitizeStreamChunk is called with boundary transition Inner→DMZ  
**Then**: INNER_ONLY taint is stripped and content is redacted  
**Expected Result**: INNER_ONLY removed from Taints, content replaced with `[INNER_DATA_REDACTED]`  
**Spec Reference**: arch-v1.md L298 (inner-data stripping), L284 (auto-strip on Inner→DMZ/outer)
