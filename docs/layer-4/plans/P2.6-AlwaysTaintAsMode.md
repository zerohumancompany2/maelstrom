## Parent Requirements

From `arch-v1.md L751` - DataSource taintMode options:
1. `inheritFromXattr` - Read taints from filesystem xattr (default)
2. `alwaysTaintAs=INNER_ONLY` - Always apply specified taints regardless of storage
3. `none` - No automatic tainting

From `arch-v1.md L1092` - DataSource taintMode in ChartDefinition:
1. taintMode can be set at DataSource configuration level
2. `alwaysTaintAs` takes a taint value (e.g., `INNER_ONLY`)
3. Overrides any existing taints on read operations

From `arch-v1.md L764-767` - On Read workflow:
1. Read tool calls `sys:datasources.getTaints(path)`
2. Security service immediately attaches those taints to resulting `tool_result` or injected ContextBlock Message
3. Existing `prepareContextForBoundary` + per-block `taintPolicy` redacts/drops/audits as configured

## Acceptance Criteria

1. DataSource with `alwaysTaintAs=INNER_ONLY` returns `["INNER_ONLY"]` from GetTaints regardless of file contents
2. DataSource with `alwaysTaintAs` option ignores xattr/object tags and always returns configured taints
3. DataSource with `taintMode=none` returns empty taint list from GetTaints
4. `alwaysTaintAs` can specify multiple taints (e.g., `alwaysTaintAs=INNER_ONLY,SECRET`)
