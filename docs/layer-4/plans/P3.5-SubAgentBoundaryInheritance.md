From `arch-v1.md L265-270` - Boundary Model:
- Inner: Full read/write to sensitive application-context slices. Prompt injection surface is zero because no untrusted Messages ever enter an inner Chart.
- DMZ: Mediated access. Tools/sub-agents are wrapped; outputs are sanitized.
- Outer: Untrusted ingress only. Can only talk to DMZ agents via mail; cannot directly read/write inner data.

## Acceptance Criteria

1. Sub-agent inherits parent boundary when spawned with same or stricter boundary
2. Sub-agent cannot elevate to a less strict boundary than parent (dmz→inner forbidden)
3. Data returned from sub-agent is tainted with sub-agent boundary marker
4. Attempting boundary elevation triggers taint_violation event

## Test Descriptions

### Test 1: TestSubAgentBoundary_InheritsParent

**Given**: Parent agent at inner boundary spawns sub-agent with inner boundary  
**When**: Sub-agent is created and boundary is validated  
**Then**: Sub-agent inherits parent's inner boundary successfully  
**Expected Result**: Sub-agent boundary matches parent boundary (inner→inner allowed)  
**Spec Reference**: arch-v1.md L1780 (stricter-only inheritance, inner→dmz allowed)

### Test 2: TestSubAgentBoundary_CannotElevate

**Given**: Parent agent at dmz boundary attempts to spawn sub-agent with inner boundary  
**When**: Sub-agent creation is attempted with elevation to inner  
**Then**: Sub-agent creation is rejected with boundary elevation error  
**Expected Result**: dmz→inner transition is forbidden, error returned  
**Spec Reference**: arch-v1.md L1780 (dmz→inner forbidden)

### Test 3: TestSubAgentBoundary_ReturnDataTainted

**Given**: Sub-agent at dmz boundary completes and returns data to parent at inner boundary  
**When**: Return data is passed through boundary validation  
**Then**: Data is tainted with DMZ boundary marker before reaching parent  
**Expected Result**: Returned data carries ["DMZ"] taint indicating sub-agent boundary origin  
**Spec Reference**: arch-v1.md L280 (sub-agent result uses allowedOnExit), L1780 (boundary inheritance)

### Test 4: TestSubAgentBoundary_ViolationOnElevation

**Given**: Sub-agent at outer boundary attempts to return inner-boundary data to parent  
**When**: Return data contains inner-boundary taints not allowed for outer boundary  
**Then**: taint_violation event is emitted to dead-letter queue  
**Expected Result**: Violation event recorded, data blocked from crossing boundary  
**Spec Reference**: arch-v1.md L286 (violation emits taint_violation event), L1780 (stricter-only inheritance)
