## Test Descriptions

### Test 1: TestInMemory_TagOnWrite

**Given**: A new InMemory DataSource instance  
**When**: TagOnWrite is called with path "/workspace/test.txt" and taints ["WORKSPACE", "TOOL_OUTPUT"]  
**Then**: Taints stored in internal map with path as key  
**Expected Result**: Taints stored in memory, path used as key for lookup  
**Spec Reference**: arch-v1.md L796 (inMemoryWorkspace), L771 (TagOnWrite workflow)

### Test 2: TestInMemory_GetTaints

**Given**: An InMemory DataSource with taints previously stored for path "/workspace/test.txt"  
**When**: GetTaints is called with the path, then with unknown path "/workspace/other.txt"  
**Then**: Returns stored taints for known path, empty list for unknown path  
**Expected Result**: Taints retrieved from memory correctly, empty list for unknown path  
**Spec Reference**: arch-v1.md L777-778 (getTaints call), L796 (in-memory workspace)

### Test 3: TestInMemory_Concurrent

**Given**: An InMemory DataSource with multiple goroutines accessing it  
**When**: Multiple goroutines call TagOnWrite and GetTaints concurrently on different paths  
**Then**: No race conditions, all operations complete successfully  
**Expected Result**: Thread-safe operations, no race conditions detected by -race flag  
**Spec Reference**: arch-v1.md L796 (fast lookup requirement implies concurrency safety)
