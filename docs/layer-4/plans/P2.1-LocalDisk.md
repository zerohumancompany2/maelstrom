## Test Descriptions

### Test 1: TestLocalDisk_TagOnWrite

**Given**: A LocalDisk DataSource with a valid path  
**When**: TagOnWrite is called with path "/tmp/test.txt" and taints ["PII", "TOOL_OUTPUT"]  
**Then**: xattr `user.maelstrom.taints` is set on the file with JSON value `["PII","TOOL_OUTPUT"]`  
**Expected Result**: xattr set successfully with correct namespace and JSON encoding  
**Spec Reference**: arch-v1.md L772 (xattr format), L771 (TagOnWrite call)

### Test 2: TestLocalDisk_GetTaints

**Given**: A file with xattr `user.maelstrom.taints=["PII","SECRET"]` already set  
**When**: GetTaints is called with the file path  
**Then**: Returns `["PII", "SECRET"]` as decoded taint list  
**Expected Result**: xattr read and JSON decoded correctly, original taint list returned  
**Spec Reference**: arch-v1.md L777-778 (getTaints call and taint attachment)

### Test 3: TestLocalDisk_SidecarFallback

**Given**: A LocalDisk DataSource on a filesystem that does not support xattr  
**When**: TagOnWrite is called and xattr operation fails  
**Then**: Sidecar file created at path+".maelstrom" with JSON taint data  
**Expected Result**: Sidecar contains valid JSON with taints array, GetTaints reads from sidecar  
**Spec Reference**: arch-v1.md L796 (fallback sidecar), implementation-assessment.md L223

### Test 4: TestLocalDisk_ValidateAccess_Allowed

**Given**: A LocalDisk DataSource with allowedForBoundary=["inner", "dmz"]  
**When**: ValidateAccess is called with boundary "dmz"  
**Then**: No error returned, access allowed  
**Expected Result**: Boundary in allowed list, operation succeeds  
**Spec Reference**: arch-v1.md L789 (ValidateAccess interface), L760 (allowedForBoundary config)

### Test 5: TestLocalDisk_ValidateAccess_Denied

**Given**: A LocalDisk DataSource with allowedForBoundary=["inner"]  
**When**: ValidateAccess is called with boundary "outer"  
**Then**: Error returned indicating boundary mismatch  
**Expected Result**: Access denied with error message containing boundary information  
**Spec Reference**: arch-v1.md L789 (ValidateAccess interface), L267-269 (boundary capabilities)
