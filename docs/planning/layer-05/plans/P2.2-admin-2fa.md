## Test Descriptions

### Test 1: TestAdminService_2FARequired

**Given**: An AdminService instance and an admin command request  
**When**: AdminCommand is called with invalid or missing 2FA token  
**Then**: Command rejected with authentication error  
**Expected Result**: Invalid token returns error, all admin commands gated by 2FA  
**Spec Reference**: arch-v1.md L467 (2FA-gated), L485 (authToken parameter)

### Test 2: TestAdminService_OuterBoundaryOnly

**Given**: An AdminService instance with agents at different boundaries (outer, DMZ, inner)  
**When**: AdminCommand is called targeting non-outer boundary agents  
**Then**: Command rejected for inner/DMZ boundary agents  
**Expected Result**: Inner boundary agents rejected, DMZ boundary agents rejected, only outer boundary agents allowed  
**Spec Reference**: arch-v1.md L467 (outer-only), L261-270 (Boundary Model)

### Test 3: TestAdminService_TokenCreation

**Given**: An AdminService instance with AuthManager configured  
**When**: Token creation requested for valid user  
**Then**: Token created and expires after configured duration  
**Expected Result**: Tokens created for valid users, tokens expire correctly, expired tokens rejected  
**Spec Reference**: arch-v1.md L467 (2FA-gated), L485 (authToken parameter)
