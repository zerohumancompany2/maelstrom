# P4.9: ContextBlock TaintPolicy

## Phase Overview

Implements per-block `taintPolicy` in ContextBlock for proactive redaction control.

## Spec References

- **L1802**: Added per-block `taintPolicy` in ContextBlock (proactive redaction)
- **L2402**: TaintPolicy definition: `enforcement` (strict/redact/audit), `allowedOnExit`, `redactRules`
- **L295**: Per-block `taintPolicy` overrides global settings
- **L779**: `prepareContextForBoundary` + per-block `taintPolicy` redacts/drops/audits as configured

## Test Specifications

### Test 1: ContextBlock_TaintPolicy_AttachedSuccessfully

**Behavior**: ContextBlock can be created with an attached `taintPolicy` that specifies enforcement mode, allowed exit taints, and redaction rules.

**Acceptance Criteria**:
- ContextBlock struct includes optional `TaintPolicy` field
- TaintPolicy can be set with `enforcement=redact`, `allowedOnExit=["USER_SUPPLIED"]`, and `redactRules` array
- TaintPolicy attached at ContextBlock creation persists through serialization
- Per-block policy takes precedence over ChartDefinition global policy

### Test 2: ContextBlock_TaintPolicy_OVERRIDES_GLOBAL

**Behavior**: When ContextBlock has explicit `taintPolicy`, it overrides the ChartDefinition-level global policy for that block.

**Acceptance Criteria**:
- Global policy set to `enforcement=strict` on ChartDefinition
- ContextBlock created with `taintPolicy.enforcement=audit`
- When block is processed for LLM prompt, audit mode applies (not strict)
- Verification: taint violations are logged but not blocked for this block
