# P4.11: ContextBlock Audit

## Phase Overview

Implements audit mode for ContextBlock taint violations, allowing policy violations to be logged without blocking.

## Spec References

- **L779**: `prepareContextForBoundary` + per-block `taintPolicy` redacts/drops/audits as configured
- **L1316**: SHALL redact/drop/audit per-block `taintPolicy` before LLM prompt
- **L1517**: Redact, drop, or audit per-block `taintPolicy` before LLM prompt assembly
- **L1857**: Taint tracking strips/audits for prompt injection mitigation
- **L2402**: TaintPolicy `enforcement` field supports `strict/redact/audit` modes

## Test Specifications

### Test 1: ContextBlock_Audit_LOGS_VIOLATIONS

**Behavior**: ContextBlock with `taintPolicy.enforcement=audit` logs taint violations without blocking LLM prompt assembly.

**Acceptance Criteria**:
- ContextBlock marked with `taints=["SECRET"]` and `taintPolicy.enforcement=audit`
- When block crosses to boundary where SECRET is forbidden, violation is logged
- Block content still included in LLM prompt (not dropped/redacted)
- Audit log entry contains: block ID, taints, boundary, timestamp, violation type

### Test 2: ContextBlock_Audit_CONTINUES_EXECUTION

**Behavior**: Audit mode allows ChartRuntime execution to continue despite taint violations, unlike strict mode which would block.

**Acceptance Criteria**:
- ContextBlock with `taintPolicy.enforcement=audit` has forbidden taint for target boundary
- `prepareContextForBoundary` completes successfully (returns no error)
- LLM prompt assembly proceeds with block included
- Violation emitted to dead-letter queue as `TaintViolation` event (L2408)
- Chart state machine continues to next state without interruption
