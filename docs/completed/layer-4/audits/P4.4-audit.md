## Critical Issues

### Issue 1: Test Name Mismatch with Gap G9

**Severity**: MEDIUM

**Description**: The test names in the plan do not match the Gap G9 specification exactly.

**Gap G9 expects:**
1. `TestContextMap_TokenBudgeting`
2. `TestContextMap_EvictionSummarize`
3. `TestContextMap_EvictionTruncate`
4. `TestContextMap_EvictionError`

**Plan has:**
1. `TestContextMapTokenBudget_EnforcesLimit`
2. `TestContextMapTokenBudget_EvictsLowestPriority`
3. `TestContextMapTokenBudget_ReservesSystemBlocks`
4. `TestContextMapTokenBudget_CompressesOnOverflow`

**Impact**: Deviation from Gap G9 may cause tracking issues.

**Resolution**: Update Gap G9 to reflect the more descriptive test names in the plan, OR update the plan to match Gap G9 exactly.

---

### Issue 2: Missing Test for `eviction: truncate` and `eviction: error`

**Severity**: HIGH

**Description**: Gap G9 specifies tests for `EvictionTruncate` and `EvictionError`, but the plan only has one test for compression (`CompressesOnOverflow`).

**arch-v1.md L211** specifies three eviction strategies:
```
eviction: summarize | truncate | error
```

**Current plan coverage:**
- ✅ `summarize` - Test 4 covers compression
- ❌ `truncate` - No explicit test
- ❌ `error` - No explicit test

**Impact**: Two of three eviction strategies are not tested.

**Resolution**: Add tests for `truncate` and `error` eviction strategies, or consolidate into a parameterized test.

---

### Issue 3: Test 2 References L212 for Priority, But L212 is About `priority: int`

**Severity**: LOW

**Description**: Test 2 references `arch-v1.md L212 (priority: assembly order)` but L212 only shows `priority: int` in the schema.

**Actual line content (arch-v1.md L212):**
```
    priority: int             # assembly order
```

**Analysis**: The comment "# assembly order" is on L212, so the reference is technically correct but could be clearer.

**Resolution**: Consider adding a comment explaining the priority semantics (lower number = higher priority).

1. **Test name mismatch with Gap G9** - Must align with Gap G9 or update Gap G9
