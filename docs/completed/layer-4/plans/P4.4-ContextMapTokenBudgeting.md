## Acceptance Criteria

1. ContextMap assembly enforces token budget limit across all blocks
2. When over budget, lowest priority blocks are evicted first
3. System-required blocks (priority 0) are never evicted
4. Blocks with `eviction: summarize` are compressed before eviction

**Scope Note**: Eviction strategies `truncate` and `error` are out of scope for this phase. Only `summarize` eviction with compression is implemented.

## Test Descriptions

### Test 1: TestContextMap_TokenBudgetEnforcement

**Given**: A ContextMap with 3 blocks totaling 5000 tokens and a budget limit of 4000 tokens  
**When**: AssembleWithBudget is called with budget 4000  
**Then**: Total tokens in assembled prompt are ≤ 4000, blocks are evicted or truncated to fit  
**Expected Result**: Token budget limit is strictly enforced  
**Spec Reference**: arch-v1.md L432 (token budgeting), L1544 (budgeting and eviction)

### Test 2: TestContextMap_EvictionPriorityOrder

**Given**: A ContextMap with blocks at priorities 1, 5, 10, 15 and budget exceeded  
**When**: AssembleWithBudget is called with insufficient budget  
**Then**: Block with priority 15 (lowest) is evicted first, then 10, then 5, preserving higher priority blocks  
**Expected Result**: Eviction order follows priority (highest priority = lowest number = evicted last)  
**Spec Reference**: arch-v1.md L212 (priority: assembly order), L1544 (eviction)

### Test 3: TestContextMap_SystemBlockPreservation

**Given**: A ContextMap with system block at priority 0 and user blocks at priorities 5, 10 and budget exceeded  
**When**: AssembleWithBudget is called with budget that can only fit system block  
**Then**: System block (priority 0) is always retained, user blocks are evicted  
**Expected Result**: System-required blocks are never evicted regardless of budget pressure  
**Spec Reference**: arch-v1.md L212 (priority), L1544 (eviction with system preservation)

### Test 4: TestContextMap_EvictionSummarize

**Given**: A ContextMap with block at priority 5 having `eviction: summarize` and budget exceeded  
**When**: AssembleWithBudget is called with insufficient budget  
**Then**: Block is compressed/summarized before eviction is attempted, reducing token count  
**Expected Result**: Compression is attempted before eviction for blocks with summarize strategy  
**Spec Reference**: arch-v1.md L213 (compressor), L211 (eviction: summarize)
