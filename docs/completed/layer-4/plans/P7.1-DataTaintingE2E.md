## Parent Requirements

From `arch-v1.md L277-287` - Data Tainting Workflow:
1. Attach taints at Message creation (Gateway → user, Orchestrator → tool_result, MemoryService → injected block, Heartbeat)
2. Attach taints at ContextBlock assembly (Security calls `applyBlockTaints`)
3. Attach taints on data leaving runtime (Mail, stream chunks, snapshot, sub-agent result) — uses `allowedOnExit`
4. Security Service propagates taints on copy/read/write (L283)
5. Inner → DMZ/outer: auto-strip or block on forbidden taints (L284)
6. Runtime guard: violation emits `taint_violation` event to dead-letter (L286)

From `arch-v1.md L276` - Security Invariant:
- "All data entering the runtime is tainted at the border. No untainted information exists inside the runtime."

From `arch-v1.md L1312-1315` (SHALL statements):
- SHALL attach taints at Message creation and file read
- SHALL propagate taints on every copy, read, and write operation
- SHALL enforce `allowedOnExit` on all Mail, stream chunks, snapshots, and sub-agent returns

## Test Descriptions

### Test 1: TestE2E_DataTainting_MessageCreationToExit

**Given**: A complete runtime with sys:gateway, sys:security, sys:observability services running  
**When**: User sends message through Gateway → message created with `USER_SUPPLIED` taint → message routed to DMZ agent → agent processes and attempts to send response with `INNER_ONLY` taint to outer boundary  
**Then**: 
- Original message has `USER_SUPPLIED` taint attached at Gateway (L278)
- Taint propagates through Mail system to DMZ agent inbox
- Response with `INNER_ONLY` taint is blocked from reaching outer boundary (L284)
- Violation logged to dead-letter queue via sys:observability (L286)
- Outer recipient receives redacted or empty message
**Expected Result**: Full chain of taint attachment → propagation → enforcement verified end-to-end  
**Spec Reference**: arch-v1.md L277-287 (taint workflow), L1312-1315 (SHALL statements)

### Test 2: TestE2E_DataTainting_AllowedExitPropagation

**Given**: A DMZ agent with taintPolicy allowing `TOOL_OUTPUT` on exit (arch-v1.md L210-213)  
**When**: DMZ agent calls tool → tool_result created with `TOOL_OUTPUT` taint → result injected into session → agent sends mail to outer agent containing tool result  
**Then**:
- Tool result has `TOOL_OUTPUT` taint attached at Orchestrator (L278)
- Taint propagates through ContextMap assembly (L279)
- Mail with `TOOL_OUTPUT` taint allowed to cross DMZ→outer boundary (L280)
- Outer agent receives mail with taint preserved
- No violation logged (taint is in `allowedOnExit`)
**Expected Result**: Allowed taints propagate through full chain without blocking  
**Spec Reference**: arch-v1.md L277-281 (attachment points), L280 (allowedOnExit), L1315 (SHALL enforce allowedOnExit)
