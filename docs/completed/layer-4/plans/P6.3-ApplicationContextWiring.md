## Acceptance Criteria

1. ApplicationContext.Get returns taints along with value for stored data
2. ApplicationContext.Set stores taints with value for later retrieval

4. ApplicationContext provides namespace isolation for different runtimes

## Test Descriptions

### Test 1: TestApplicationContext_GetTaints_ReturnsStoredTaints

**Given**: A kernelApplicationContext with key "user-data" set with value and taints `["PII", "USER_SUPPLIED"]`  
**When**: Get is called with key "user-data" and callerBoundary "inner"  
**Then**: Returns value and taints `["PII", "USER_SUPPLIED"]`  
**Expected Result**: Taints match stored values, returned alongside value  
**Spec Reference**: arch-v1.md L384 (Get returns taints), L383-387 (ApplicationContext interface)

### Test 2: TestApplicationContext_SetTaints_StoresWithKey

**Given**: A kernelApplicationContext  
**When**: Set is called with key "secret-data", value, taints `["SECRET", "INNER_ONLY"]`, and callerBoundary "inner"  
**Then**: Value and taints stored together, retrievable via Get  
**Expected Result**: Subsequent Get returns same value with same taints  
**Spec Reference**: arch-v1.md L385 (Set stores taints), L276 (security invariant - all data tainted)

### Test 3: TestApplicationContext_BoundaryFilter_HidesForbidden

**Given**: A kernelApplicationContext with key "inner-secret" set with taints `["INNER_ONLY", "SECRET"]`  
**When**: Get is called with callerBoundary "outer"  
**Then**: Either returns error or returns value with forbidden taints filtered/hidden  
**Expected Result**: Outer boundary cannot access INNER_ONLY data, boundary enforcement applied  
**Spec Reference**: arch-v1.md L261-270 (boundary model), L284 (inner→outer auto-strip), L384 (callerBoundary parameter)

### Test 4: TestApplicationContext_Namespace_Isolation

**Given**: Multiple kernelApplicationContext instances with different namespaces (e.g., "agent-1", "agent-2")  
**When**: Set and Get called with same key in different namespaces  
**Then**: Each namespace has isolated data, no cross-contamination  
**Expected Result**: Namespace isolation works, different namespaces separated  
**Spec Reference**: arch-v1.md L386 (Namespace method), L770-771 (isolation hook)
