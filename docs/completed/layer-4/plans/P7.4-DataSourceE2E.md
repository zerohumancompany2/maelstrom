## Parent Requirements

From `arch-v1.md L757-767` - DataSource Write/Read Workflows:

**On Write** (L757-761):
1. Write action calls `sys:datasources.tagOnWrite(path, incomingTaints)`
2. For localDisk: sets xattr `user.maelstrom.taints=PII,TOOL_OUTPUT` (or JSON-encoded)
3. For object stores: sets object tags/metadata
4. For workspaces: updates sidecar manifest for fast lookup

**On Read** (L764-767):
1. Read tool calls `sys:datasources.getTaints(path)`
2. Security service immediately attaches those taints to resulting `tool_result` or injected ContextBlock Message
3. Existing `prepareContextForBoundary` + per-block `taintPolicy` redacts/drops/audits as configured

## Test Descriptions

### Test 1: TestE2E_DataSource_WriteReadTaintCycle

**Given**: 
- LocalDisk DataSource configured with `xattrNamespace: user.maelstrom` (arch-v1.md L754)
- Agent workspace at `/agents/testagent/workspace` (L749-750)
- File to write: `sensitive_data.txt` with content containing PII
- Taints to attach: `["PII", "TOOL_OUTPUT"]`

**When**: 
1. Agent writes file via tool → `tagOnWrite(path, ["PII", "TOOL_OUTPUT"])` called (L757)
2. Agent reads file via tool → `getTaints(path)` called (L764)
3. Read result injected as `tool_result` Message

**Then**:
- xattr `user.maelstrom.taints` set on file with JSON-encoded `["PII","TOOL_OUTPUT"]` (L759)
- `getTaints` returns `["PII", "TOOL_OUTPUT"]` from xattr (L765)
- Security service attaches taints to `tool_result` Message (L766)
- Message.Metadata.Taints contains `["PII", "TOOL_OUTPUT", "TOOL_OUTPUT"]` (union with existing)
- Full write → tag → read → attach cycle verified end-to-end
**Expected Result**: DataSource write/read cycle preserves and propagates taints through tool_result  
**Spec Reference**: arch-v1.md L757-767 (DataSource write/read workflows), L1312 (SHALL attach taints at file read)

### Test 2: TestE2E_DataSource_ContextMapBoundaryEnforcement

**Given**: 
- DMZ agent with ContextMap containing ContextBlock from DataSource (arch-v1.md L205-223)
- ContextBlock source: `memoryService` reading from workspace file (L229-234)
- File has taints: `["INNER_ONLY"]` from previous write
- ContextBlock has `taintPolicy.redactMode: dropBlock` for `INNER_ONLY` (L217)
- Agent in DMZ boundary attempting LLM call

**When**: Agent assembles ContextMap → reads file from DataSource → `prepareContextForBoundary` invoked  
**Then**:
- Read tool calls `getTaints(path)` → returns `["INNER_ONLY"]` (L764-765)
- Security attaches `INNER_ONLY` taint to ContextBlock Message (L766)
- `prepareContextForBoundary` detects forbidden taint for DMZ boundary (L293)
- ContextBlock dropped entirely per `dropBlock` policy (L294)
- LLM receives prompt without the tainted ContextBlock
- No `INNER_ONLY` data in assembled prompt
- Violation logged to sys:observability (audit trail)
**Expected Result**: DataSource taints flow through ContextMap assembly with boundary enforcement  
**Spec Reference**: arch-v1.md L764-767 (DataSource read workflow), L292-296 (ContextMap enforcement), L217-218 (dropBlock policy)
