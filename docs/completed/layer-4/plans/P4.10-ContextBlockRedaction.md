# P4.10: ContextBlock Redaction

## Phase Overview

Implements redaction rules for ContextBlock content based on taint policy configuration.

## Spec References

- **L268**: DMZ outputs sanitized with PII redaction, length limits, schema validation
- **L298**: Every outgoing chunk passes through DMZ sanitizers (PII redaction, length caps, schema validation)
- **L1523**: SHALL NOT allow inner data to reach outer/DMZ LLM prompts without redaction
- **L1745**: Proactive + reactive tainting enables automatic redaction at boundary crossing
- **L1802**: Per-block `taintPolicy` in ContextBlock (proactive redaction)
- **L1862**: Tainting + `allowedOnExit` enforcement; redaction rules for data exfiltration

## Test Specifications

### Test 1: ContextBlock_Redaction_AppliesPIIRules

**Behavior**: ContextBlock with PII taint is redacted according to configured redaction rules before LLM prompt assembly.

**Acceptance Criteria**:
- ContextBlock marked with `taints=["PII"]` and `taintPolicy.redactRules` configured
- Redaction rule specifies pattern replacement (e.g., email addresses → `[REDACTED]`)
- When `prepareContextForBoundary` is called, PII content is replaced per rules
- Redacted content logged to audit trail with original taint markers preserved in metadata

### Test 2: ContextBlock_Redaction_DROPS_FORBIDDEN_TAINTS

**Behavior**: ContextBlock with forbidden taints for target boundary is dropped entirely from ContextMap.

**Acceptance Criteria**:
- ContextBlock marked with `taints=["INNER_ONLY"]` attempting to cross to DMZ boundary
- Boundary policy marks `INNER_ONLY` as forbidden (not in `allowedOnExit`)
- When `prepareContextForBoundary` is called, entire block is removed from prompt
- Drop action logged with reason: "forbidden taint INNER_ONLY for DMZ boundary"
