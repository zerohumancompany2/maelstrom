## Test Descriptions

### Test 1: TestPersistence_RefusesBoundaryViolation

**Given**: A Persistence layer with TaintPolicy enforcement enabled  
**When**: A write is attempted with data containing taints ["INNER_ONLY", "SECRET"] that violate the boundary policy for outer boundary  
**Then**: Write is blocked and error returned indicating taint policy violation  
**Expected Result**: Error contains "taint policy violation" or similar, write does not persist data  
**Spec Reference**: arch-v1.md L285 (Persistence refuses writes that would violate policy)

### Test 2: TestPersistence_EnforcesTaintPolicy

**Given**: A Persistence layer with TaintPolicy configured with enforcement="strict"  
**When**: Multiple write operations are attempted with varying taint configurations  
**Then**: TaintPolicy is enforced on all writes regardless of data type or source  
**Expected Result**: All writes validated against TaintPolicy, violations rejected consistently  
**Spec Reference**: arch-v1.md L285 (taints stored with data; Persistence refuses writes that would violate policy)

### Test 3: TestPersistence_AllowsCompliantWrites

**Given**: A Persistence layer with TaintPolicy allowing ["USER_SUPPLIED", "TOOL_OUTPUT"] taints  
**When**: A write is attempted with data containing taints ["USER_SUPPLIED"]  
**Then**: Write succeeds and taints are stored alongside the data  
**Expected Result**: Data persisted successfully, taints retrievable via GetTaints  
**Spec Reference**: arch-v1.md L285 (taints stored with data)
