## Test Descriptions

### Test 1: TestTaintEngine_AttachTaint_Mail

**Given**: A Mail struct with existing taints `["USER_SUPPLIED"]`  
**When**: AttachTaint is called with new taints `["TOOL_OUTPUT"]`  
**Then**: Mail.Metadata.Taints contains `["USER_SUPPLIED", "TOOL_OUTPUT"]`  
**Expected Result**: Taints are merged (union operation), existing taints preserved  
**Spec Reference**: arch-v1.md L278 (Message creation), L283 (taint propagation)

### Test 2: TestTaintEngine_AttachTaint_Map

**Given**: A `map[string]interface{}` with data but no `_taints` key  
**When**: AttachTaint is called with taints `["PII", "SECRET"]`  
**Then**: Map receives `_taints` key with value `["PII", "SECRET"]`  
**Expected Result**: Map values are not modified, only `_taints` key added  
**Spec Reference**: arch-v1.md L273 (taint sets), L279 (ContextBlock assembly)

### Test 3: TestTaintEngine_AttachTaint_Nested

**Given**: A nested map structure with multiple levels  
**When**: AttachTaint is called with taints `["INNER_ONLY"]`  
**Then**: All nested maps receive the taint at their respective levels  
**Expected Result**: Taints attached at root and all nested map levels  
**Spec Reference**: arch-v1.md L283 (taint propagation on copy/read/write)

### Test 4: TestTaintEngine_AttachTaint_Empty

**Given**: An empty taint list or nil object  
**When**: AttachTaint is called with empty taints or nil object  
**Then**: Empty taint list does not modify object; nil object returns error  
**Expected Result**: Appropriate error handling for edge cases  
**Spec Reference**: arch-v1.md L276 (security invariant - all data tainted)
