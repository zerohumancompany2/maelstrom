### Test 1: TestBoundaryImmutability_HotReload

**Given**: Chart with boundary set to "inner" loaded in ChartRegistry  
**When**: Hot-reload is triggered with new ChartDefinition that attempts to change boundary to "dmz"  
**Then**: Boundary remains "inner" and reload succeeds with boundary preserved  
**Expected Result**: Boundary is immutable across hot-reload operations  
**Spec Reference**: arch-v1.md L126-127, L1783 (boundary is NEVER migratable)

### Test 2: TestBoundaryImmutability_VersionUpgrade

**Given**: Chart with boundary set to "outer" and version "1.0.0"  
**When**: Version upgrade to "2.0.0" with migrationPolicy.onVersionChange set to "deepHistory"  
**Then**: Boundary remains "outer" after version upgrade completes  
**Expected Result**: Boundary is immutable across version changes regardless of migration policy  
**Spec Reference**: arch-v1.md L893 (metadata.boundary — NEVER migratable)

### Test 3: TestBoundaryImmutability_MigrationPolicy

**Given**: Chart with boundary set to "dmz" and migrationPolicy configured  
**When**: Migration operation attempts to include boundary in contextTransform or onVersionChange  
**Then**: Error is returned rejecting boundary modification attempt  
**Expected Result**: migrationPolicy cannot affect boundary field under any configuration  
**Spec Reference**: arch-v1.md L1781-1783 (migrationPolicy excludes boundary from migratable properties)
