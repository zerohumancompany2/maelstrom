## Acceptance Criteria

1. Inner → DMZ transition strips INNER_ONLY and PII taints automatically
2. Inner → outer transition strips all sensitive taints, only EXTERNAL allowed
3. Outer → inner transition blocks on PII/SECRET taints, allows USER_SUPPLIED with EXTERNAL taint added
4. DMZ → inner transition blocks on forbidden taints, allows valid DMZ data
5. Same boundary transitions pass through without modification
6. All boundary transitions occur via Mail only (enforced by sys:security)

## Test Descriptions

### Test 1: TestTransition_InnerToDMZ

**Given**: Mail with boundary=inner and taints=["INNER_ONLY", "PII", "TOOL_OUTPUT"] transitioning to dmz  
**When**: EnforceTransition is called with source=inner, target=dmz  
**Then**: INNER_ONLY and PII taints are stripped, TOOL_OUTPUT preserved  
**Expected Result**: Resulting taints=["TOOL_OUTPUT"], no error returned  
**Spec Reference**: arch-v1.md L284 (Inner → DMZ/outer: auto-strip forbidden taints)

### Test 2: TestTransition_InnerToOuter

**Given**: Mail with boundary=inner and taints=["SECRET", "PII", "TOOL_OUTPUT", "EXTERNAL"] transitioning to outer  
**When**: EnforceTransition is called with source=inner, target=outer  
**Then**: SECRET, PII, TOOL_OUTPUT stripped, only EXTERNAL preserved  
**Expected Result**: Resulting taints=["EXTERNAL"], all sensitive taints removed  
**Spec Reference**: arch-v1.md L284 (Inner → DMZ/outer: auto-strip forbidden taints), L269 (outer is ingress only)

### Test 3: TestTransition_OuterToInner

**Given**: Mail with boundary=outer and taints=["USER_SUPPLIED", "PII"] transitioning to inner  
**When**: EnforceTransition is called with source=outer, target=inner  
**Then**: PII blocked with error, USER_SUPPLIED allowed with EXTERNAL taint added  
**Expected Result**: Error returned for PII, or if only USER_SUPPLIED: taints=["USER_SUPPLIED", "EXTERNAL"]  
**Spec Reference**: arch-v1.md L284 (block on forbidden taints), L267 (inner has zero prompt injection surface)

### Test 4: TestTransition_DMZToInner

**Given**: Mail with boundary=dmz and taints=["TOOL_OUTPUT", "SECRET"] transitioning to inner  
**When**: EnforceTransition is called with source=dmz, target=inner  
**Then**: SECRET blocked with error, TOOL_OUTPUT allowed  
**Expected Result**: Error returned for SECRET taint, valid DMZ data allowed through  
**Spec Reference**: arch-v1.md L284 (block on forbidden taints), L268 (DMZ mediated access)

### Test 5: TestTransition_SameBoundary

**Given**: Mail with boundary=dmz and taints=["TOOL_OUTPUT"] transitioning to dmz  
**When**: EnforceTransition is called with source=dmz, target=dmz  
**Then**: No modification to taints, no error  
**Expected Result**: Resulting taints=["TOOL_OUTPUT"], pass-through behavior  
**Spec Reference**: arch-v1.md L290 (transitions across boundaries - same boundary is not a transition)

### Test 6: TestTransition_ForbiddenTaints

**Given**: Mail with boundary=outer and taints=["SECRET", "INNER_ONLY"] transitioning to inner  
**When**: EnforceTransition is called with source=outer, target=inner  
**Then**: Both SECRET and INNER_ONLY blocked with error listing forbidden taints  
**Expected Result**: Error includes both forbidden taint names, transition blocked  
**Spec Reference**: arch-v1.md L284 (block on forbidden taints), L286 (violation emits taint_violation event)
