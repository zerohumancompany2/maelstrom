### Test 2: TestNamespaceIsolate_FilterData

**Given**: An IsolatedView for runtime `agent-123` with boundary `dmz` and data containing taints `["INNER_ONLY", "TOOL_OUTPUT"]`  
**When**: FilterData is called on the IsolatedView  
**Then**: Data with `INNER_ONLY` taint excluded, `TOOL_OUTPUT` data accessible  
**Expected Result**: Forbidden data filtered based on boundary, allowed data accessible  
**Spec Reference**: arch-v1.md L770 (isolated executor with DataSource policy), L294 (forbidden taint filtering)

**Given**: An IsolatedView created with operation `write`  
**When**: Operation is retrieved from the IsolatedView  
**Then**: Operation type is `write`, view has write-appropriate restrictions  
**Expected Result**: Different operations have different view restrictions (read vs write)  
**Spec Reference**: arch-v1.md L771 (all FS ops through mediated tool layer), spec-extraction.md L787-793 (DataSource interface)
