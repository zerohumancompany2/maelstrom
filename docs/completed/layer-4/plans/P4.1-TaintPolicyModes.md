## Parent Requirements

From `arch-v1.md L209, L916` - Enforcement Modes:
1. `strict` - Block forbidden taints entirely
2. `redact` - Replace forbidden values with replacement text
3. `audit` - Allow but log violations
4. `allowedOnExit` - Define allowed taints for outbound data
5. Redact rules application per taint category

From `arch-v1.md L914-919` - ChartDefinition taintPolicy schema:
```yaml
taintPolicy:
  enforcement: strict | redact | audit
  allowedOnExit: ["PII", "SECRET", "TOOL_OUTPUT"]
  redactRules:
    - taint: "SECRET"
      replacement: "[REDACTED]"
```

From `spec-extraction.md L348-350` - TaintPolicy Constraints:
- `enforcement` defines mode (strict/redact/audit)
- `allowedOnExit` enforced on all outbound data paths
- Taints not in allowedOnExit list are blocked/redacted at boundaries

## Acceptance Criteria

1. Policy enforcement in `strict` mode blocks forbidden taints with error
2. Policy enforcement in `redact` mode replaces forbidden values with replacement text
3. Policy enforcement in `audit` mode allows data through but logs violations
4. `allowedOnExit` policy filters taints on outbound data
5. Multiple `redactRules` applied correctly per taint category

## Test Descriptions

### Test 1: TestPolicyEnforcement_Strict

**Given**: A data object with taints `["SECRET", "PII"]` and a TaintPolicy with `enforcement: strict` and `allowedOnExit: ["TOOL_OUTPUT"]`  
**When**: EnforcePolicy is called with the data and policy  
**Then**: An error is returned blocking the data, no data passes through  
**Expected Result**: Forbidden taints cause immediate error, data rejected  
**Spec Reference**: arch-v1.md L209 (strict mode), L916 (enforcement modes)

### Test 2: TestPolicyEnforcement_Redact

**Given**: A data object with taints `["SECRET"]` and a TaintPolicy with `enforcement: redact` and `redactRules: [{taint: "SECRET", replacement: "[REDACTED]"}]`  
**When**: EnforcePolicy is called with the data and policy  
**Then**: The SECRET-tainted values are replaced with `[REDACTED]`, data passes through  
**Expected Result**: Forbidden values replaced, block structure preserved  
**Spec Reference**: arch-v1.md L209 (redact mode), L918-919 (redactRules)

### Test 3: TestPolicyEnforcement_Audit

**Given**: A data object with taints `["PII"]` and a TaintPolicy with `enforcement: audit` and `allowedOnExit: ["TOOL_OUTPUT"]`  
**When**: EnforcePolicy is called with the data and policy  
**Then**: Data passes through unchanged, violation is logged to audit trail  
**Expected Result**: No blocking, violation recorded for observability  
**Spec Reference**: arch-v1.md L209 (audit mode), spec-extraction.md L286 (violation emission)

### Test 4: TestPolicyEnforcement_AllowedOnExit

**Given**: A data object with taints `["PII", "TOOL_OUTPUT", "SECRET"]` and a TaintPolicy with `allowedOnExit: ["TOOL_OUTPUT"]`  
**When**: EnforcePolicy is called with the data and policy in redact mode  
**Then**: Only TOOL_OUTPUT taints remain, others stripped or blocked based on enforcement mode  
**Expected Result**: allowedOnExit acts as whitelist for outbound taints  
**Spec Reference**: arch-v1.md L916 (allowedOnExit), spec-extraction.md L348-350

### Test 5: TestPolicyEnforcement_MultipleRedactRules

**Given**: A data object with taints `["SECRET", "PII"]` and a TaintPolicy with `enforcement: redact` and multiple `redactRules` for each taint  
**When**: EnforcePolicy is called with the data and policy  
**Then**: Each taint category redacted with its specific replacement text  
**Expected Result**: SECRET → "[REDACTED]", PII → "[PERSONAL_INFO]"  
**Spec Reference**: arch-v1.md L918-919 (redactRules array), L218-221 (per-block redactRules)
