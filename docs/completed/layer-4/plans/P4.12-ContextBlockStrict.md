# P4.12: ContextBlock Strict

## Phase Overview

Implements strict enforcement mode for ContextBlock taint policy, blocking any taint violations at boundary crossings.

## Spec References

- **L2402**: TaintPolicy `enforcement` field with `strict/redact/audit` modes (strict blocks violations)
- **L294**: Any block with forbidden taints is redacted or dropped before LLM call
- **L1317**: SHALL enforce `allowedOnExit` on Mail, streams, snapshots, sub-agent returns
- **L1518**: Enforce `allowedOnExit` on all Mail, stream chunks, snapshots, and sub-agent returns
- **L2408**: TaintViolation event emitted to dead-letter queue on policy violation

## Test Specifications

### Test 1: ContextBlock_Strict_BLOCKS_VIOLATIONS

**Behavior**: ContextBlock with `taintPolicy.enforcement=strict` blocks LLM prompt assembly when forbidden taints are detected.

**Acceptance Criteria**:
- ContextBlock marked with `taints=["SECRET"]` and `taintPolicy.enforcement=strict`
- Boundary does not include SECRET in `allowedOnExit` list
- When `prepareContextForBoundary` is called, operation returns error
- Error message indicates: "strict enforcement blocked block with forbidden taint: SECRET"
- Block is excluded from LLM prompt assembly

### Test 2: ContextBlock_Strict_EMITS_VIOLATION_EVENT

**Behavior**: Strict enforcement emits TaintViolation event to dead-letter queue when blocking occurs.

**Acceptance Criteria**:
- ContextBlock with `taintPolicy.enforcement=strict` violates boundary policy
- `prepareContextForBoundary` blocks the violation
- TaintViolation event emitted with: blockID, taints, boundary, policy, timestamp
- Event routed to dead-letter queue for later analysis
- Chart execution may continue but without the blocked block in context
