## Test Descriptions

### Test 1: TestSecurityService_HandleMail_BoundaryTransition

**Given**: A SecurityService with mail crossing from outer to inner boundary with taints `["USER_SUPPLIED"]`  
**When**: HandleMail is called with the mail  
**Then**: Mail is validated, EXTERNAL taint is added, and mail is routed to target  
**Expected Result**: Mail.Metadata.Taints contains `["USER_SUPPLIED", "EXTERNAL"]`, boundary updated to inner  
**Spec Reference**: arch-v1.md L1303 (ValidateAndSanitize), L290 (Mail-only transitions)

### Test 2: TestSecurityService_TaintPropagate_NestedStructures

**Given**: A SecurityService and nested map structure with multiple levels containing maps and slices  
**When**: TaintPropagate is called with taints `["PII", "SECRET"]`  
**Then**: All nested maps receive the taints via `_taints` key, taints are merged with existing  
**Expected Result**: Root and all nested maps have complete taint set, no duplicates  
**Spec Reference**: arch-v1.md L1305 (TaintPropagate), L283 (taint propagation on copy/read/write)

### Test 3: TestSecurityService_NamespaceIsolate_FilteredView

**Given**: A SecurityService and data containing items with different agentIDs  
**When**: NamespaceIsolate is called with runtimeId "agent-123" and operation "read"  
**Then**: Only items matching agentID "agent-123" are returned in isolated view  
**Expected Result**: IsolatedView contains filtered data, operation tracked  
**Spec Reference**: arch-v1.md L1306 (NamespaceIsolate), L770-771 (isolation hook)

### Test 4: TestSecurityService_CheckTaintPolicy_EnforcementModes

**Given**: A SecurityService and data with taints `["PII", "INNER_ONLY"]` and policy with enforcement mode  
**When**: CheckTaintPolicy is called with targetBoundary "outer" and policy mode "strict"  
**Then**: Returns false (blocked) for forbidden taints in strict mode, true for allowed  
**Expected Result**: Policy enforcement mode respected, boundary considered in decision  
**Spec Reference**: arch-v1.md L1307 (CheckTaintPolicy), L209 (enforcement modes), L916 (policy fields)

### Test 5: TestSecurityService_ReportTaints_CompleteMap

**Given**: A SecurityService with taints tracked for runtimeId "agent-456" across multiple objects  
**When**: ReportTaints is called with runtimeId "agent-456"  
**Then**: TaintMap returned with runtimeId as key, mapping objects to their taints  
**Expected Result**: All taints for runtime tracked and retrievable, empty map for unknown runtime  
**Spec Reference**: arch-v1.md L1308 (ReportTaints), L285 (taints stored with data)
