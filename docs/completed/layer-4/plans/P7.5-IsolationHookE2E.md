## Test Descriptions

### Test 1: TestE2E_IsolationHook_DirectSyscallBlocked

**Given**: 
- Agent running in isolated namespace (arch-v1.md L770)
- Workspace mounted via DataSource with policy enforcement
- Tool layer mediation active for all FS operations
- sys:observability running for violation reporting

**When**: Agent attempts direct filesystem access via syscall (e.g., `open("/etc/passwd")` bypassing tool layer)  
**Then**:
- Direct syscall blocked by isolation mechanism (L771)
- Error returned to agent: "direct syscall not allowed - use mediated tool layer"
- Violation logged to sys:observability dead-letter queue (L286)
- Violation contains: agent ID, attempted path, isolation policy violated
- No filesystem access granted
**Expected Result**: Direct syscalls blocked with violation reported end-to-end  
**Spec Reference**: arch-v1.md L770-771 (isolation hook), L1316 (SHALL use DataSource for all FS access), L286 (violation to dead-letter)

### Test 2: TestE2E_IsolationHook_ToolLayerMediation

**Given**: 
- Agent in namespace `agent:researcher` with workspace `/agents/researcher/workspace`
- Another agent in namespace `agent:analyst` with workspace `/agents/analyst/workspace`
- DataSource policy: `isolation: strict` (arch-v1.md L515)
- Tool `readFile` registered with boundary-aware access

**When**: 
1. Researcher agent calls `readFile` tool to read `/agents/researcher/workspace/data.txt`
2. Researcher agent attempts to read `/agents/analyst/workspace/secret.txt` (cross-namespace)

**Then**:
- Legitimate read (own workspace) → routed through tool layer → DataSource.getTaints called → file read succeeds (L764)
- Cross-namespace read → NamespaceIsolate detects boundary violation (L1305)
- Tool layer blocks access: "namespace isolation violation"
- Violation logged to sys:observability with details
- Analyst's file not accessible to Researcher
- Each tool executes in own namespace per isolation policy (L515)
**Expected Result**: Tool layer mediates FS ops with namespace isolation enforcement  
**Spec Reference**: arch-v1.md L770-771 (isolation hook), L515-519 (orchestrator isolation), L1305 (NamespaceIsolate), L1316 (SHALL use DataSource)

func (s *SecurityService) NamespaceIsolate(runtimeID, operation string) (IsolatedView, error) {
    namespace := s.getNamespace(runtimeID)
    allowedPaths := s.getAllowedPaths(namespace)
    
    if !pathWithinAllowedPaths(operation.Path, allowedPaths) {
        return IsolatedView{}, errors.New("namespace isolation violation")
    }
    
    return IsolatedView{Namespace: namespace, Paths: allowedPaths}, nil
}
