### Test 2: TestNamespaceIsolate_BlocksCrossNamespaceAccess

**Given**: An IsolatedView for runtime `agent-123` with boundary `dmz`  
**When**: Access is attempted to data from runtime `agent-456`  
**Then**: Access is blocked with namespace violation error  
**Expected Result**: Error returned indicating cross-namespace access denied  
**Spec Reference**: arch-v1.md L781 (isolated executor with DataSource policy), arch-v1.md L294 (boundary enforcement)

### Test 3: TestNamespaceIsolate_FiltersDataByBoundaryAndTaint

**Given**: An IsolatedView for runtime `agent-123` with data containing taints `["INNER_ONLY", "TOOL_OUTPUT"]` and boundary `dmz`  
**When**: GetViewData is called on the IsolatedView  
**Then**: Data with `INNER_ONLY` taint excluded, `TOOL_OUTPUT` data accessible  
**Expected Result**: Filtered data returned based on boundary and taint policies  
**Spec Reference**: arch-v1.md L782 (mediated tool layer), arch-v1.md L294 (forbidden taint filtering)
